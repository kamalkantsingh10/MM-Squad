# Correct Course - Validation Checklist

## Pre-Change Validation

- [ ] Current sprint state reviewed and understood
- [ ] Change rationale documented and approved by user
- [ ] Dependency analysis consulted for migration order impact
- [ ] No dependency violations introduced by proposed changes

## Change Execution Validation

- [ ] All deferred modules reassigned to future milestones
- [ ] All added modules assigned to current milestone
- [ ] All label updates applied correctly
- [ ] Every change has a GitLab comment with rationale

## Post-Change Validation

- [ ] Updated sprint capacity calculated and reported
- [ ] No orphaned dependencies (consumer without its shared service)
- [ ] Sprint timeline impact assessed
- [ ] README dashboard updated if needed
