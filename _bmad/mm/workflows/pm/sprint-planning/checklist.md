# Sprint Planning - Validation Checklist

## Pre-Sprint Validation

- [ ] Migration order reviewed and confirmed with user
- [ ] All target modules have existing GitLab Issues
- [ ] No dependency violations (shared services included before consumers)
- [ ] Sprint date range is reasonable for the scope

## Milestone Validation

- [ ] Milestone created successfully in GitLab
- [ ] Milestone has correct start and end dates
- [ ] Milestone description includes sprint goal and module list

## Assignment Validation

- [ ] All target module Issues assigned to the milestone
- [ ] Issues assigned in dependency order
- [ ] No module assigned to multiple active milestones
- [ ] Sprint capacity calculated and reported

## Capacity Check

- [ ] Total complexity units within team capacity
- [ ] No single High-complexity module without buffer time
- [ ] Sprint scope aligns with migration order priorities
